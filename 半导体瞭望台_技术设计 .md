# 半导体瞭望台 · 技术设计

## 技术栈
- 前端：React + TypeScript + Vite + Tailwind CSS + Recharts（图表）+ Lucide React（图标）
- 后端：Node.js + Express
- 数据：各平台 JSON 接口（fetch 解析，非 HTML 爬虫）
- 缓存：内存 Map（TTL 30~60 秒）
- 部署：前端 Vercel / 后端 Railway（示例）

## 数据模型

### EtfQuote（ETF 实时行情）
```typescript
interface EtfQuote {
  name: string;           // "半导体ETF"
  code: string;           // "159813"
  price: number;          // 1.730
  change: number;         // -0.01
  changePercent: number;  // -0.57
  high: number;
  low: number;
  volume: number;         // 成交量
  history30d: number[];   // 近30日收盘价
  periods: {
    "5d": number;
    "1m": number;
    "3m": number;
    "1y": number;
  };
}
```

### IndexQuote（指数行情）
```typescript
interface IndexQuote {
  name: string;           // "国证芯片"
  code: string;           // "980017"
  price: number;
  changePercent: number;
  history30d?: number[];  // 可选，部分指数无历史
}
```

### MemoryPrice（存储芯片价格）
```typescript
interface MemoryPrice {
  name: string;           // "DRAM 合约价"
  period: string;         // "2026Q2"
  changePercent: number;  // 45
  prevPeriod: string;     // "2026Q1"
  prevChangePercent: number; // 55
  status: "hot" | "warning" | "cold";
  statusLabel: string;    // "🔥 动能充足"
  source: string;         // "TrendForce"
}
```

### FundFlow（资金流向）
```typescript
interface FundFlow {
  northBound: {
    today: number;        // 当日净流入（负数为流出）
    "5d": number[];       // 近5日数据
  };
  etfShare: {
    current: number;      // 当前份额
    change1m: number;     // 近1月变化
    change1mPercent: number;
  };
}
```

### SignalItem（信号看板）
```typescript
interface SignalItem {
  id: string;             // "memory_momentum"
  name: string;           // "存储涨价动能"
  level: "green" | "yellow" | "red";
  levelLabel: string;     // "🟢 充足"
  summary: string;        // "DRAM Q2 环比涨幅维持 45%"
  suggestion: string;     // "持有"
  detail: string;         // 展开后的详细说明
  triggeredAt: string;    // "2026-05-30"
}
```

### ApiResponse（统一响应）
```typescript
interface ApiResponse<T> {
  code: number;           // 0 = 成功
  message: string;
  data: T;
  updatedAt: string;      // ISO 8601
  stale?: boolean;        // true = 数据可能滞后
}
```

## 错误码对照表

| 错误码 | 含义 | 触发场景 |
|--------|------|---------|
| 0 | 成功 | 一切正常 |
| 1001 | 上游限流，返回缓存 | 东方财富 API 返回 403/429 |
| 1002 | 飞书未配置，返回 Mock | 缺少飞书凭证或表格 ID |
| 1003 | 部分接口失败，部分降级 | 北向资金限流但 ETF 份额正常 |
| 1004 | 依赖数据缺失，信号不准确 | 存储价格未录入 |
| 5001 | ETF 数据源完全不可用 | 东方财富 API 挂了且无缓存 |
| 5002 | 指数数据源完全不可用 | 全部指数接口失败 |
| 5003 | 飞书凭证无效 | app_id/app_secret 错误 |
| 5004 | 资金流向完全不可用 | 全部资金接口失败 |
| 5005 | 信号引擎异常 | 代码运行时错误 |

## 核心流程

### 1. 页面加载流程
```
用户打开首页
  → 前端并行请求 5 个接口（/api/etf/quote, /api/index/quotes, /api/memory/price, /api/fund/flow, /api/signal/board）
  → 后端检查缓存
      ├─ 命中且未过期 → 直接返回缓存
      └─ 未命中或过期 → 调用对应 Service 抓取 → 解析 JSON → 写入缓存 → 返回
  → 前端渲染五大卡片
  → 启动 30 秒轮询（仅 ETF 行情和指数）
  → 启动 5 分钟轮询（产业价格、资金流向、信号看板）
```

### 2. 东方财富数据抓取流程
```
GET /api/etf/quote?code=159813
  → cache.get("etf_159813")
      ├─ 命中且 TTL < 30s → 返回
      └─ 未命中
          → fetch("http://push2.eastmoney.com/api/qt/stock/get?secid=0.159813")
          → 解析返回的 JSONP / JSON
          → 提取 price, change, changePercent, volume 等字段
          → cache.set("etf_159813", data, 30s)
          → 返回 ApiResponse<EtfQuote>
```

### 3. 飞书表格读取流程（产业价格）
```
GET /api/memory/price
  → cache.get("memory_price")
      ├─ 命中且 TTL < 5min → 返回
      └─ 未命中
          → feishu.readSheet({ appToken, sheetId, range: "A1:F10" })
          → 解析表格行数据 → 映射为 MemoryPrice[]
          → cache.set("memory_price", data, 5min)
          → 返回 ApiResponse<MemoryPrice[]>
```

### 4. 信号引擎计算流程
```
定时任务（node-cron，每小时）
  → 读取当前缓存中的所有指标数据
  → 逐条执行规则判断
      ├─ 规则1：dramQ2Change >= 40 ? green : dramQ2Change >= 20 ? yellow : red
      ├─ 规则2：etfShareChange1m < -50000000 && priceChange5d > 10 ? yellow
      ├─ 规则3：soxChange3d < -5 && chipIndexChange3d > 5 ? red
      └─ 规则4：priceChange5d > 15 && etfShareChange1m < 0 ? red
  → 生成 SignalItem[]
  → cache.set("signal_board", signals, 5min)
```

## 开发环境代连

- 开发：`npm run dev` 同时启动 Vite 前端（:3000）和 Express 后端（:3001）
- 代理：Vite `server.proxy` 将 `/api/*` 代理到 `http://localhost:3001`
- 生产：VITE_API_BASE 指向后端域名，或 Nginx 反代

## 数据源方案备注

- 主链路：自建 Express 拉取各平台 JSON（东方财富、Yahoo Finance）
- 数据：免费第三方金融 API（不稳定，建议加缓存 + Mock 兜底）
- 不推荐：微博 OAuth、HTML 爬虫（易被封且维护成本高）
- 飞书表格：需申请飞书应用凭证（app_id + app_secret），MVP 阶段可先用本地 JSON 替代
