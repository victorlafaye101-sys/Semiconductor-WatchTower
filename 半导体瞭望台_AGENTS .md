# 半导体瞭望台 · 开发指令

## 项目概述
使用 React + TypeScript + Vite + Tailwind CSS 开发前端；
使用 Node.js + Express 开发后端，聚合 ETF 实时行情、存储芯片价格、资金流向、信号看板等多维数据，辅助半导体投资决策。
目标部署：前端 Vercel，后端 Railway。

**我没有编程背景，请 AI 每步给出可直接运行的完整代码，遇到配置问题主动说明操作步骤。**

---

## 开发规范
- 使用 TypeScript，类型定义必须与「数据模型」章节保持一致
- 使用函数式组件 + Hooks，禁止 class 组件
- 样式使用 Tailwind CSS，保持简洁
- 可复用组件：`ETFCard`、`IndexGrid`、`PriceFuel`、`FundFlow`、`SignalBoard`、`Layout`

## 代码风格
- 组件名 PascalCase，函数 camelCase
- 接口路径：`/api/etf/quote`、`/api/index/quotes`、`/api/memory/price`、`/api/fund/flow`、`/api/signal/board`
- **禁止**在前端直接 fetch 东方财富 / Yahoo Finance 等第三方域名（必须经后端中转，避免跨域被封）

## 设计要求
- 暗色科技风：深色背景 `#0f172a`，青绿高亮 `#10b981`，参考 Claude.ai 首页信息密度
- 卡片效果：毛玻璃（`backdrop-blur`）、细边框、悬浮微上浮动效
- 数字用等宽字体；涨跌幅颜色遵循 A 股习惯（**涨红跌绿**）
- 桌面端 3 列卡片网格，移动端 1 列堆叠
- ETF 行情卡片最大最醒目；信号看板红/黄/绿灯做视觉强调
- 单卡失败只影响该卡片，不拖垮整页（显示对应错误文案）
- 页脚注明：「仅供个人投资研究 · 数据来源：东方财富 / TrendForce · 非投资建议」

## MVP 功能范围（P0 先上线）

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | ETF 实时行情 | 159813 价格、涨跌幅、近5日/1月/3月/1年涨幅、30日迷你折线图 |
| P0 | 指数联动看板 | 国证芯片 980017、费城半导体 SOX、科创 50 |
| P0 | 信号看板 | 4 条红绿灯规则，可点击展开详情面板 |
| P1（后续） | 产业价格燃料 | DRAM / NAND / HBM 状态，飞书表格手动录入 |
| P1（后续） | 资金流向 | 北向资金 + ETF 份额变化 |
| P2（更晚） | 历史趋势大图 | 点击指标查看近1年走势 |

**不做清单（MVP 明确排除）：** 用户登录、真实交易下单、股票个股行情、自动抓取 TrendForce、邮件/短信推送、移动端 App、机器学习模型。

## 注意事项
- 上游请求加合理 `User-Agent`、`Referer`（参照各平台公开文档）
- **禁止**将飞书 `app_id/app_secret`、数据库 URL 等敏感信息提交到公开 GitHub（统一放 `.env`，`.gitignore` 必须包含 `.env`）
- 开发阶段先用 Mock 数据跑通 UI，再逐个接入真实接口
- 页脚注明：仅供个人投资研究，非投资建议
- 前端开发代理：Vite `server.proxy` 将 `/api/*` 转发到 `http://localhost:3001`

---

## 开发顺序（请严格按此推进）
1. 搭页面框架 + Mock 数据，确认浏览器能看到五张卡片
2. 逐一实现后端接口，每个接口完成后验证数据返回正确
3. 前端替换 Mock 为真实接口
4. 生成 Vercel（前端）+ Railway（后端）部署配置文件

## 验收标准（每个接口完成后检查）
- [ ] 接口返回的 JSON 结构与「数据模型」一致
- [ ] 东方财富限流时，卡片显示缓存数据而不是白屏
- [ ] 10 分钟内反复刷新，后端不会每次都打上游（缓存生效）
- [ ] 某接口挂掉，其他四张卡片仍正常展示
- [ ] 信号规则修改后，看板颜色正确变化
